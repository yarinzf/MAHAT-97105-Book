# פרק 1 – איך לחשוב במבחן

## המטרה
לא להתחיל לכתוב קוד לפני שמבינים מה באמת שואלים.

### ארבע שאלות לפני כל פתרון
1. מהו מבנה הנתונים?
2. האם מותר לשנות אותו?
3. מה מחזירים?
4. האם צריך לעבור על כל האיברים?

## זיהוי מהיר
| אם כתוב | כנראה צריך |
|---|---|
| Queue | Temp Queue אם אסור לשנות |
| Stack | Temp Stack |
| Node | מעבר עם next |
| true/false | flag |
| count | מונה |
| max/min | משתנה מקסימום/מינימום |

## טעות נפוצה
להתחיל לכתוב קוד לפני שמזהים את התבנית.

## כלל זהב
